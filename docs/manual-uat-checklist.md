# FactLog 수동 테스트 시나리오

**목적:** 제출 직전, 실제 사용자가 FactLog를 처음 열어 핵심 플로우를 끝까지 수행할 수 있는지 수동으로 확인한다.  
**대상 버전:** v0.1.0 MVP  
**테스트 일자:** ________  
**테스터:** ________

## 1. 테스트 전 준비

### 실행 조건
- 백엔드 API가 실행 중이어야 한다.
  - `uvicorn factlog_ml.api:app --reload`
- 프론트엔드 개발 서버가 실행 중이어야 한다.
  - `cd web && npm run dev`
- NASA 검증 요약 산출물이 최신 상태여야 한다.
  - `python scripts/run_validation.py --data-dir data/raw/nasa-cmapss --subset FD001`
  - `python scripts/export_validation_summary.py`

### 사전 확인
- 브라우저에서 앱 첫 화면이 열리는지 확인
- 콘솔 에러가 치명적으로 쌓이지 않는지 확인
- `artifacts/validation/factlog_validation_summary.md`가 존재하는지 확인

## 2. 합격 기준

- 최소 1회 `수기 입력 → 분석 → 근거 확인 → 조치 저장 → 대시보드 반영`이 성공해야 한다.
- 최소 1회 `CSV 업로드 → 분석 반영`이 성공해야 한다.
- 근거 설명, 유사 사례, 검증 요약이 화면 또는 산출물에서 확인 가능해야 한다.
- 치명적 blocker가 0건이어야 한다.

## 3. 시나리오 1: 첫 진입과 빈 상태 확인

**목적:** 첫 사용자가 길을 잃지 않는지 확인

### 절차
1. 앱 첫 화면을 연다.
2. 온보딩 또는 샘플 분석 유도 문구가 보이는지 확인한다.
3. 입력 패널, 분석 결과 패널, 조치 기록 패널, 대시보드가 한 화면 안에서 구분되는지 본다.

### 기대 결과
- "처음이라면 샘플 분석부터"에 해당하는 온보딩 문구가 보인다.
- 빈 화면이 아니라 다음 행동이 무엇인지 알 수 있다.

## 4. 시나리오 2: 수기 입력 기반 핵심 플로우

**목적:** MVP의 가장 중요한 사용자 플로우를 직접 검증

### 절차
1. 기본 샘플값 또는 수기 입력값으로 분석을 실행한다.
2. 분석 완료 메시지가 노출되는지 확인한다.
3. 이상 점수, 이상 여부, 근거 설명이 표시되는지 확인한다.
4. 유사 사례가 최대 3건까지 표시되는지 본다.
5. 조치 내용, 담당자, 결과 메모를 입력한다.
6. `조치 기록 저장`을 클릭한다.
7. 저장 성공 메시지와 대시보드 반영 여부를 확인한다.

### 기대 결과
- 분석이 정상 완료된다.
- 근거 설명이 숫자 또는 조건과 함께 자연어로 보인다.
- 조치 저장 후 최근 이력 또는 조치 건수가 증가한다.
