# ADR-001: NASA C-MAPSS 이진 분류 기준선 채택

**일자:** 2026-04-03
**상태:** Accepted

## 맥락 (Context)
FactLog MVP는 2~3일 안에 "설비 이상 감지와 근거 제시"를 제품 플로우로 보여줘야 한다. 동시에 채용 과제 성격상 공개 데이터 기반 정량 검증 수치를 제시해야 한다. 따라서 모델 선택은 최고 성능보다 재현 가능성, 설명 용이성, 화면 연결 가능성을 우선해야 했다.

## 고려한 선택지

### 선택지 A: NASA C-MAPSS RUL 회귀 문제를 그대로 사용
- 장점: 원 데이터셋의 전형적인 사용 방식과 가깝다.
- 장점: 잔여수명 예측이라는 메시지가 기술적으로 강해 보인다.
- 단점: MVP의 "이상 여부 판단" 플로우와 직접 연결하기 어렵다.
- 단점: 발표에서 회귀 지표를 현장 판단 언어로 설명하기가 더 어렵다.

### 선택지 B: NASA C-MAPSS를 `RUL <= 30` 이진 분류 문제로 변환
- 장점: "이상/정상" UI 및 대시보드와 바로 연결된다.
- 장점: 정확도, 정밀도, 재현율, F1, ROC-AUC로 수치를 설명하기 쉽다.
- 단점: RUL 회귀보다 정보 손실이 있다.
- 단점: threshold 30 cycle 선택의 임의성이 일부 존재한다.

### 선택지 C: 초기부터 AI허브 데이터까지 제품 내부 완전 통합
- 장점: 글로벌 벤치마크와 국내 제조 데이터를 한 제품 안에서 비교할 수 있다.
- 장점: 발표 메시지가 더 풍부해진다.
- 단점: 스키마 정합성과 전처리 비용이 커져 MVP 일정이 흔들린다.
- 단점: 핵심 리스크인 NASA 기준선 검증이 늦어진다.

## 결정 (Decision)
선택지 B를 채택했다. NASA C-MAPSS `FD001`를 MVP 메인 검증 데이터셋으로 사용하고, `RUL <= 30 cycle`을 이상 상태로 간주하는 이진 분류 기준선을 만들었다. 모델은 `LogisticRegression(class_weight="balanced")`를 사용하고, AI허브 데이터는 이 단계에서 제품 내부 완전 통합이 아니라 보조 검증 대상으로만 유지한다.

## 결과 (Consequences)
- 긍정적: MVP 플로우와 정량 검증 수치를 빠르게 연결할 수 있다.
- 긍정적: 모델, 설명, 대시보드, 향후 유사 사례 검색이 같은 "이상 여부" 언어로 정렬된다.
- 부정적: 잔여수명 회귀 문제의 세밀한 정보를 버린다.
- 부정적: threshold 30 cycle은 향후 도메인 검토로 조정될 수 있다.
- 리스크: AI허브 데이터에서 같은 기준이 그대로 통하지 않을 수 있다.

## 되돌릴 조건 (Reversal Triggers)
- AI허브 보조 검증에서 이진 분류 기준이 현저히 불안정할 때
- 제품 요구가 "이상 여부"보다 "잔여수명 예측" 중심으로 바뀔 때
- 시계열 딥러닝 도입이 MVP 이후의 핵심 차별화 포인트로 확정될 때
